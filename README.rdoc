== Service Desk (light version)

http://ipsilonrb.herokuapp.com

Функционал проекта планируемый к реализации:

Модуль Service Desk
Модуль представляет собой единую точку контакта для всех потребителей услуг.
Специалисты через данный модуль осуществляют регистрацию различных типов запросов,
разрешают максимально возможное число инцидентов на первой линии поддержки и
назначают исполнителей на инцидент в случае невозможности решить его на первой линии.

Заявки и инциденты
Основной модуль системы.
Обеспечивается прием заявок по проблемам и инцидентам, возможность отслеживания жизненого цикла, автоэксалация и т.п.

База знаний
Модуль системы для сбора информации о часто возникающих инцидентах и их самостоятельного решения.

Учёт активов
Модуль учета активов позволяет вести каталог оборудования и создавать инциденты с привязкой к конкретному оборудованию.

Система уведомлений
Система предполагает уведомления об изменениях в заявках с отчетом о проведенных изменениях в заявке.
Уведомления о вновь создаваемых заявках. Уведомления о назначении исполнителя заявки и т.д.

Данный функционал призван реализовать следующие процессы:
Процесс управления инцидентами
Процесс управления проблемами
Процесс управления конфигурациями
Процесс управления изменениями и т.д

Реализованный функционал:
1 модуль работы с заявками:
  для пользователя доступно:
    1 создание заявки (с привязкой к активу при наличии)
    2 возможность добавления сообщений в заявку
    3 прикрипление файлов к заявке
    4 dashborad со списком всех заявок пользователя (по-умолчанию не отображаются закрытые)
    5 уведомления по почте о смене статуса заявки
    6 после закрытия заявки - возможность оценить работу по заявке
    7 фильтрация заявок по статусам
  для администратора/диспетчера:
    1 полное управление всеми заявками
    2 установка статуса, приоритета и исполнителя по заявке
    3 фильтрация заявок по статусам
    4 возможность добавления сообщений в заявку
    5 прикрипление файлов к заявке
    6 dashborad со списком всех заявок (по-умолчанию не отображаются закрытые)
  для специалиста доступно тоже, что и пользователю, но видит заявки которые назначенны ему, может менять статус заявки

2 модуль База знаний:
  для пользователя, специалиста возможность просмотра инструкций и статей из базы знаний
  для администратора/диспетчера - просмотр, редактирование и удаление

3 модули Организация, Департамент, Активы, Приоритеты - являются служебными, позволяют обеспечивать вывод дополнительной
информации по заявке и пользователю. Настраивает и привязываеет значения администратор/диспетчер

4 поиск по заявкам, базе знаний для пользователя/специалиста
